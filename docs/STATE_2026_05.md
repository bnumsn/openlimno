## WEDM v0.2 case schema (added fields)

`case.bbox` — `[lon_min, lat_min, lon_max, lat_max]` EPSG:4326.

| `data.*` block | Source | Required fields | Enum-validated fields |
|---|---|---|---|
| `dem` | Copernicus | `uri` (string) | — |
| `lulc` | WorldCover | `uri` | `year ∈ {2020, 2021}`, `version ∈ {v100, v200}`, class_km2 keys ∈ 11 LCCS codes |
| `soil` | SoilGrids | `uri`, `lat`, `lon` | `depths` ⊂ 6 SoilGrids depths, `statistic` ∈ 5 stats |
| `watershed` | HydroSHEDS | `uri`, `pour_lat`, `pour_lon` | `region` ∈ 9 continent codes, `level` ∈ 1-12 |
| `species_occurrences` | GBIF | `uri`, `scientific_name`, `usage_key` | `match_type` ∈ {EXACT, FUZZY, HIGHERRANK, NONE}, `confidence` 0-100 |
| `climate` | Daymet / Open-Meteo | `uri`, `source` | `source` ∈ {daymet, open-meteo} |

JSON Schema `$id` deliberately retained at `0.1` to preserve the `$ref` graph
(case ↔ builtin_1d / passage / schism / studyplan sub-schemas). The "schema
version" exposed to users is the case-document language version in
`openlimno: '0.2'`.

## Defensive-design pins (per-ship discoveries)

| Ship | Bug pattern caught | Pin test |
|---|---|---|
| v0.3.0 | DEM cache key omitted bbox → different bboxes reused first cached subset | `test_request_key_differs_by_params` |
| v0.3.1 | Daymet API silently snaps pre-1980 requests to its window → CSV would be silently mislabelled | `test_daymet_rejects_pre_1980_year` |
| v0.3.2 | Cache key must distinguish (lat, lon) points | `test_open_meteo_cache_key_includes_bbox_like_params` |
| v0.3.3 | `ogr.UseExceptions()` promotes `.sbn` warning to RuntimeError → real HydroSHEDS shapefile reads abort | `test_hydrosheds_does_not_enable_global_ogr_exceptions` |
| v0.3.4 | Exact-3°-edge bbox would pull an extra zero-overlap tile (bandwidth waste) | `test_worldcover_tiles_for_bbox_exact_3deg_edge_no_extra_tile` |
| v0.3.5 | SoilGrids stores int × d_factor; missing inverse scaling = 10× wrong fractions | `test_soilgrids_parses_response_and_applies_d_factor` |
| v0.3.6 | GBIF occasionally returns null lat/lon despite `hasCoordinate=true` | `test_species_occurrence_filters_null_coordinates` |
| v0.5.0 | JSON Schema `$id` bump breaks relative `$ref` graph | (lessons-learned note above) |

## Coverage matrix (data type × region)

| Region | DEM | Q | Climate | Watershed | LULC | Soil | Species |
|---|---|---|---|---|---|---|---|
| US / Canada | cop30 | **nwis** | **daymet** (1 km) or open-meteo | hydrosheds (na) | worldcover | soilgrids | gbif |
| Europe / Russia | cop30 | (open-meteo only) | open-meteo | hydrosheds (eu/ar/si) | worldcover | soilgrids | gbif |
| China / Asia | cop30 | (open-meteo only) | open-meteo | hydrosheds (as) | worldcover | soilgrids | gbif |
| Africa / Australia / S. America | cop30 | (open-meteo only) | open-meteo | hydrosheds (af/au/sa) | worldcover | soilgrids | gbif |

Only data-type × region gap: **gauge discharge outside the US**. No
public global gauge API with NWIS-equivalent ease exists. GRDC = monthly +
registration; Chinese provincial portals require reverse-engineered crawlers
(dibiaoshui-style — out of OpenLimno's "subscription-free public APIs"
charter; tracked as v0.8.2 `openlimno-cn` plugin frame instead).

## Engineering discipline (consistently applied this arc)

- 5-round single-AI critique before every ship (substitute for the
  `triple_review.sh` that the sandbox can't run).
- Real-data smoke against live APIs before tagging.
- API-surface regression pins on `__all__` so a refactor breaks CI loudly.
- Every new fetcher carries a defensive-design pin from its 5-round
  critique (DEM bbox, Daymet pre-1980, HydroSHEDS UseExceptions, …).
- No new ruff / mypy findings vs. each ship's parent commit (except for
  the GUI dialog's E702/I001 widget-setup pattern that matches the file's
  existing `build_case_from_osm` style).
