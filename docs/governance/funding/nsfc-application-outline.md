## 立项依据 (3-4 页要点)

### 国际研究现状

- PHABSIM (USFWS 1995, Fortran 77, 已不维护) — 30 年来 IFIM 方法事实
  标准,但停止演进
- River2D (Univ. Alberta, 2010 后无更新) — 2D 单元 PHABSIM 替代
- 法美欧均面临**评估工具老化**问题:Web of Science 2020-2025 间引用
  PHABSIM 的论文中 ≥40% 提到"代码不可复现"或"数据格式过时"

### 国内研究现状

- 中国 SL/Z 712-2014《河湖生态环境需水计算规范》要求"四元组"(月最低
  / 适宜流量 / 多年平均比例 / 90% 保证率),但缺乏开源参考实现
- 长江、黄河、珠江流域生态流量评估多采用商业软件 (MIKE / Delft3D),
  数据黑箱,审查困难
- "十四五"水生态保护规划明确"加强生态流量监督",但工具链国产开源
  化几乎空白

### 待解决科学问题

1. **数据契约问题**: 如何在水文-水动力-生物多源异构数据上建立可
   复核、可追溯的开放数据模型 (WEDM)?
2. **算法等价性问题**: 自研 1D 引擎 + SCHISM 2D 适配器在 SL712
   评估上能否做到与 PHABSIM 表级等价 (≤1e-3)?
3. **本土化生境问题**: 如何把 IFIM 框架与中国"四大家鱼"漂流性卵
   产卵繁殖生态联系起来 (易伯鲁 1988 / 段中华 / 曹文宣等本土证据)?
4. **跨流域可比性问题**: 长江、黄河、澜沧江尺度差异巨大,如何让
   栖息地评估结果在多流域间可比?

## 研究内容 (4 个子课题)

### 子课题 1:WEDM 数据模型本土化 (PI 项目组)
- SL712 四元组与 WEDM 时间序列契约对齐
- 中国本土物种 HSI 库 (≥ 10 个目标物种,含中华鲟、四大家鱼、
  圆口铜鱼等)
- 监测数据格式适配:水利部 / 生态环境部 / 中科院水生所标准

### 子课题 2:水动力引擎中文文档与本土化测试
- 自研 1D 引擎中文术语对齐 (《水力学》朱泽藩术语)
- SCHISM v5.11.0 在长江中游典型断面的标定
- 与中国水利水电科学研究院河口海岸所联合验证

### 子课题 3:典型流域案例 (核心交付)
- **长江中游某支流** (建议:汉江中游或赤水河) 全流程案例
  - ADCP 实测、断面、底质、四大家鱼监测数据采集
  - SL712 四元组评估 + 不确定性量化
  - 与现有商业软件 (MIKE) 对比,差异分析
  - 本地审查员 (中科院水生所 / 长江委 / 水科院) 签字记录
- 数据公开发布 (CC-BY-4.0),长期 archived 到 ScienceDB / 中国科学院
  数据云

### 子课题 4:维护者社区建设
- 招募 ≥ 3 位国内 maintainer (水利院校 + 水利系统 + 中国科学院)
- 与中国水利学会、水利信息化分会建立长期伙伴关系
- 年度用户大会 (与水利信息化年会合办)
