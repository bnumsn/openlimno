## Concerns

**Bug risk — controller.py:817-832 (mtime cache)**
- `os.path.getmtime` resolution is 1 s on many filesystems (FAT, older ext, networked shares); a user who re-runs preprocess and re-clicks within the same second sees stale rows. Use `os.stat(...).st_mtime_ns` plus `st_size` to defeat sub-second writes and same-mtime overwrites.
- The `mtime = None` fallback is unsafe: if the file vanishes (deleted or unmounted), both the cached `None` and the new `None` compare equal → returns stale cached rows instead of surfacing the OSError. Either re-raise, or store a sentinel that never compares equal (e.g. `object()`).
- `cache_key = self._xs_parquet` is taken verbatim from the attribute — if it's stored as a relative path, the `getmtime` call resolves against the *current* CWD, which may differ from when it was first cached (Qt file dialogs often `chdir`). Normalise via `os.path.realpath` before keying.
- `os.path.getmtime` follows symlinks; a swapped symlink target with an older mtime won't invalidate. Marginal, but the docstring claims the change makes the cache correct under all overwrite scenarios.

**Test adequacy**
- The cache change has **zero** new tests. A regression where the mtime check silently no-ops (e.g. comparing `None == None`) would pass everything. Add a unit test that mutates the parquet between two `_load_xs_rows`-equivalent calls and asserts a re-read.
- `test_appimage_run_smoke.py:21-37` removes all coverage of the `OSMCaseSpec` → `build_case` path. That code is now exercised nowhere in this test module. Either add an `@pytest.mark.online` test that still calls Overpass, or you've quietly deleted regression coverage for the OSM builder.
- The binary fixtures (`cross_section.parquet`, `hsi_curve.parquet`, `mesh.ugrid.nc`) have no regeneration script or schema-version pin. The README says "auto-generated by `openlimno init-from-osm`" but offers no reproducible recipe. When the parquet schema bumps, the fixture rots silently and the smoke test asserts on outputs derived from a stale schema. Add a `make fixtures` target or a test that round-trips the fixture through the current loader and asserts column compatibility.
- `shutil.copytree(FIXTURE, out, dirs_exist_ok=True)` into a freshly `mktemp`'d dir doesn't need `dirs_exist_ok`; keeping it masks accidental fixture pollution.

**Packaging / portability — openlimno-studio.spec:13-29**
- Requirements are prose, not enforced. A clean `ubuntu-latest` GHA runner has neither `qgis` apt nor PyQt5; without an explicit `apt-get install qgis python3-pyqt5 qtbase5-dev` step in CI, this builds nowhere reproducibly. QGIS apt package versions also differ across Ubuntu codenames (jammy ships QGIS 3.22 LTR, noble ships 3.34) — pin the codename or the QGISFLOW PPA.

Add a build-time assertion in the spec body that `jsonschema.__version__` resolves to ≥ 4.18 from the venv's `site-packages` path, not `/usr/lib/python3/dist-packages`.
- Spec text claims `numpy 2.x` is required but the bundle's actual numpy ABI isn't pinned anywhere visible — risk of mixed-ABI bundle if the venv drifts.
