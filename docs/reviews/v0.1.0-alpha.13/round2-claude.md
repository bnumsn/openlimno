# Review: v0.1.0-alpha.10..e05c191

Substantive concerns only, prioritised.

## P1 — real bugs

**1. OGR fallback leaks file handles + crashes on layer-None.** `controller.py:69-75`. `ogr.Open()` returns a Dataset that's never closed; long PyQt sessions accumulate open file descriptors. Worse, `GetLayer(0)` returns `None` for malformed/multi-layer-without-default datasets, so `defn = layer.GetLayerDefn()` raises `AttributeError`. The cache wrapper at `controller.py:881` deliberately *doesn't* catch `AttributeError` (treats it as programmer bug), and round-1's new GUI try/except at `controller.py:151,189` only catches `(OSError, ValueError, RuntimeError)` — so a corrupt OGR dataset escapes both layers and surfaces as Qt's default unhandled-exception crash. Wrap in try/finally with `ds = None` and explicit `if layer is None: raise OSError(...)`.

**2. GUI exception lists diverge from cache wrapper.** `controller.py:151,189` catch `(OSError, ValueError, RuntimeError)` but the cache wrapper at `:881-890` catches `(OSError, EOFError, RuntimeError, ValueError)`. `EOFError` from a truncated parquet now escapes both `open_wua_q` and `plot_cross_section`. Pick one tuple and share it.

**3. Missing-backend RuntimeError still retried.** Acknowledged as residual #1 in `docs/reviews/v0.1.0-alpha.10/README.md` but not actually fixed in the round-1 follow-up. Three calls to `_read_wua_parquet` × 50ms backoff per call = 150ms of UI freeze on every WUA open in a stripped install. Trivial fix: dedicated `MissingBackendError(Exception)` not in the retry tuple.

**4. `show-ref` validation has a case-collision hole.** `scripts/triple_review.sh:50-54`. The regex matches `$base_lower` but `show-ref --verify "refs/heads/$BASE"` checks the original case. A user with branch `cafe` who passes `CAFE` enters the abbreviated-SHA branch (lowercase passes regex), `show-ref` against `CAFE` misses, and the script rejects a real branch. Use `$base_lower` in show-ref too, or normalise once.

## P2 — test gaps

**5. The "behaviorally new" code paths are still mocked.** Every test in `test_xs_rows_cache.py` patches `_read_wua_parquet`. Not exercised against the unmocked function:
- A real torn parquet (truncated bytes) actually raising `ArrowInvalid` and being caught by `except ValueError`. Empirical proof of the MRO claim only exists in the alpha.10 codex.md transcript, not in CI.
- OGR fallback at all (no test forces `pq = None`).
- `ds is None` branch raising `OSError`.
- `GetLayer(0) is None` AttributeError leak (issue #1).
- The new GUI try/except blocks at `controller.py:151,189` — zero tests touch them.
- `tests/unit/test_qgis_plugin_smoke.py:73` and `tests/integration/studio_workflows_smoke.py:401` call `_read_wua_parquet` expecting `[]` on empty/malformed input. With the new contract these now raise. Either stale tests or a silently-broken contract — verify.

## P3 — design / process

**6. Five rounds of `except`-tuple churn is the smell, not the fix.** Round-7 narrowed `Exception`, round-8 split `ValueError` out, round-1 duplicates the (different!) tuple in two GUI sites. Define the boundary once: a `ParquetReadError` raised by `_read_wua_parquet` for transient I/O, everything else propagates. Callers catch one type.

**7. `pq.read_table → c.to_pylist() × N → zip` materialises every column twice.** `controller.py:57-61`. For a 200k-row × 10-col cross-section that's hundreds of MB of Python objects, on the GUI thread, with no LRU cap on `_xs_rows_cache`. `to_pydict()` + per-row construction halves peak memory; `iter_batches()` bounds it. Gemini called this out and it was filed as residual but not addressed.

**8. CI gate removal is unmentioned in the round-1 commit message.** Deleting `.github/workflows/triple-review.yml` means no automated review on tag push — the methodology now relies entirely on maintainer discipline, with no enforcement that `reviews/` is populated before release. Worth a CHANGELOG note.

## Not seen
SQL/HTTP injection, integer overflow, CRS handling, real privileged-file overwrites. `os.path.realpath` + stat-read-stat handles symlink TOCTOU adequately for this threat model.
