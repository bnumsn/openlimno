# Claude review ‚Äî `v0.1.0-alpha.10..86ff7a7` (round-9 perspective)

Concerns, priority order. I treat the two GUI direct-call sites as the load-bearing finding.

**1. P0 ‚Äî `open_wua_q` / `plot_cross_section` now crash on torn parquet** (`controller.py:144`, `controller.py:174`).
Round-7 deliberately made `_read_wua_parquet` raise instead of returning `[]`. Round-8 README correctly tags this as "GUI direct-call paths bypass the cache wrapper" ‚Äî but ships it as P2. It is not P2: those handlers' `if not rows:` branches at lines 145 and 175 used to be the *only* error UX. With the new helper, a user opening a corrupt file now gets an unhandled traceback (Qt's default handler shows it, but the controller never reaches the `QMessageBox.warning(... f"Could not read {xs_path}")` at line 176). Either route both call sites through `_read_xs_rows_cached`, or wrap both raw calls in try/except and surface the error via the host's `message_bar`. Shipping this state contradicts the round-7 rationale ("torn reads silently produced empty rows that got cached") for the *cached* path while reintroducing the inverse hazard for the uncached paths.

**2. P1 ‚Äî `RuntimeError` from missing backend retried with backoff** (`controller.py:881`).
`_read_wua_parquet` raises `RuntimeError("neither pyarrow nor GDAL available...")`; the retry loop catches `RuntimeError` and burns 3 √ó `0.05*attempt s` sleep before giving up. Permanent install failure modeled as transient I/O. Round-8 README files this as residual #1 ‚Äî but the fix is one line: introduce `class _MissingBackend(Exception)` and exclude it from the retry tuple, or special-case `RuntimeError` whose message starts with the sentinel string. Two separate reviewers (Gemini, Claude in r8) flagged this; deferring is cheap-and-easy debt.

**3. P1 ‚Äî `NotImplementedError` is a `RuntimeError`** (`controller.py:881`).
`pyarrow.lib.ArrowNotImplementedError ‚Üí NotImplementedError ‚Üí RuntimeError`. Hitting an unsupported parquet encoding now triggers 3 retries instead of failing fast. Same retry-budget burn pattern as #2. Tests cover none of pyarrow's six `Arrow*Error` subclasses' actual MRO behaviour; the inline comment is the only reasoning, and r8 already shows that comment-only invariants get this wrong.

**4. P1 ‚Äî short SHA bypass still open** (`scripts/triple_review.sh:44-45`).
Codex's probe confirmed `git rev-parse --short=6 HEAD` produces a 6-char hex that `git check-ref-format --allow-onelevel` accepts. Rejecting only `^[0-9a-f]{7}$` is narrower than the round-6 rule it replaces; collision protection is now strictly weaker than alpha.9. The right shape is `^[0-9a-f]{4,11}$ AND git rev-parse resolves to commit AND not also a refname` ‚Äî keep collision protection, lose the false-rejects-on-`cafe`-branch.

**5. P2 ‚Äî OGR fallback leaks file handles, GetLayer(0) None-deref** (`controller.py:69-75`).
`ogr.Open(path)` never closed; `ds.GetLayer(0)` can return `None` for malformed datasets, and `layer.GetLayerDefn()` then raises `AttributeError`, which the retry loop classifies as programmer bug and propagates uncaught. Add `if layer is None: raise OSError(...)` and a `try/finally: ds = None` (OGR's idiomatic close).

**6. P2 ‚Äî `to_pylist()` materialisation for large XS** (`controller.py:58-61`).
`zip(*[c.to_pylist() for c in t.columns])` triples peak memory vs `t.to_pylist()`. A 200k-row √ó 12-col cross-section table goes from ~30MB Arrow buffer to ~250MB Python objects on the GUI thread. Pre-existing-ish, but the round-7 refactor was the chance to fix it.

**7. Architecture nit** ‚Äî five rounds of re-narrowing the same 6-line `except` tuple is itself the smell. The right shape: classify in the helper (raise `_TransientReadError` or `_FatalReadError`), let the cache catch one base class. Stops the bikeshed.

**8. Test gaps**: only `TypeError` is exercised for "no-retry programmer bugs"; `AttributeError`, `KeyError`, `NotImplementedError` aren't. No test for the missing-backend case (issue #2) or the OGR fallback (issue #5). No real torn-parquet test ‚Äî all mocks. A regression in any of these would pass CI.

Not seen: SQL/HTTP injection, integer overflow, CRS handling, symlink TOCTOU.
