### Architecture
- **Diff range mismatch.** Diff is labeled `v0.1.0-alpha.10..4c6ec27` but `pyproject.toml` jumps `0.1.0a10 → 0.1.0a13`. Three rounds (a.11, a.12, a.13) are bundled into one review, yet only one round's residuals are documented. Either the labeling is wrong or two rounds shipped without their own audit trail.
- **`MissingParquetBackend(RuntimeError)` ordering is load-bearing** (`controller.py:933-940`). Because `MissingParquetBackend ⊂ RuntimeError`, the `except MissingParquetBackend: raise` clause must precede `except (OSError, EOFError, RuntimeError)`. A future `ruff --fix`/sort/refactor that reorders these silently re-introduces the retry-on-permanent-failure behavior. Better: don't subclass `RuntimeError`.
- **GUI direct-call paths now lump MissingParquetBackend into a generic "Could not read" dialog** (`controller.py:182,220`). Users who hit the "no pyarrow, no GDAL" path see *"Could not read /path/to/file:\n…"* rather than *"install pyarrow"* — the whole point of the named subclass is lost at the user surface.

### Bug risk
- **`finally: ds = None` is cargo-culted** (`controller.py:111`). Inside the `try` we already build the full list before the finally fires, so the local-name reassignment runs after the materialized list (whose `feat`s held refs to `layer→ds`) is returned — the references that actually pinned the fd are released by the comprehension exit, not by the `ds = None`. The `noqa: F841` confirms it's only there to satisfy the diff. It's harmless but doesn't actually do what the comment claims.
- **8-char hex branch still bypasses the new disambiguation** (`scripts/triple_review.sh:48`). The 4-7 char path runs `rev-parse --symbolic-full-name`, but lengths 8-40 are accepted as SHAs without checking whether a real branch by that name exists. A branch literally named `cafebabe` (8 chars) is treated as a SHA, never validated as a ref.
- **`pq` and `ArrowException` import binding** (`controller.py:65-69`): if pyarrow exists but `pyarrow.lib` doesn't expose `ArrowException` (some vendor builds, conda variants), the whole `import` raises `ImportError` and falls through to the GDAL branch even though pyarrow is installed — silent backend downgrade.
