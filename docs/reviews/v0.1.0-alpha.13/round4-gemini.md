*   **Zip Integrity:** The use of `strict=False` in `zip(*[c.to_pylist() for c in t.columns])` (line 81) silently truncates data if columns are of unequal length. While rare in Parquet, `strict=True` (Python 3.10+) would be a safer default for a scientific modeling platform.

### 4. Performance
*   **Memory Inefficiency:** `_read_wua_parquet` still performs a full row-wise conversion via `c.to_pylist()` and `zip`. *   **GUI Thread Blocking:** `open_wua_q` and `plot_cross_section` call `_read_wua_parquet` directly on the main thread. A large reach file or a slow network mount will freeze the UI during the $O(N \cdot M)$ conversion process.
