## Bug risk

**Arrow exceptions outside the catch list crash the GUI** — `controller.py:923`. Cache catches `(OSError, EOFError, RuntimeError, ValueError)`. But `ArrowCapacityError` inherits only from `Exception`, and `ArrowKeyError` from `LookupError` — both escape uncaught. The runtime probe in `codex.md` listed this MRO; the maintainer used it to vindicate `ArrowInvalid` and stopped looking.

**`OSError` for malformed-but-stable parquet retries 3× pointlessly** — `controller.py:84,88`. `if ds is None` and `if layer is None` raise `OSError`, which is in the transient list. A permanently corrupt file burns 150 ms of GUI time before failing. Same issue the `MissingParquetBackend` fix addressed, just unfixed for OGR.

**`git rev-parse` shell-flag injection** — `scripts/triple_review.sh:47,75`. `git rev-parse --symbolic-full-name "$BASE"` and `git rev-parse --verify "${BASE}^{commit}"` lack `--`. A `BASE` starting with `-` is interpreted as a flag. Pass `--` before `"$BASE"` in both calls.

**Only `ANTHROPIC_API_KEY` stripped** — `scripts/triple_review.sh:185`. `ANTHROPIC_AUTH_TOKEN` and `ANTHROPIC_BASE_URL` are also honored by `claude -p` and would re-introduce the API path the change is supposed to forbid. Strip the family, not one var.

**Hex-name validation still soft for 8+ chars** — `scripts/triple_review.sh:62`. The disambiguation only runs for 4–7-char hex. An abbreviated SHA of 8–40 hex chars still passes the SHA regex without checking it's actually a commit. Run `git rev-parse --verify` for *all* hex matches, not just 4–7.

## Performance
